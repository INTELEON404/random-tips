## স্লাইড ২ — ICT নিরাপত্তা কী?

### 🔍 সংজ্ঞা

> **ICT নিরাপত্তা (Information and Communication Technology Security)** হলো এমন একটি ব্যবস্থা যা তথ্য, সিস্টেম এবং নেটওয়ার্ককে অননুমোদিত অ্যাক্সেস, ব্যবহার, প্রকাশ, পরিবর্তন বা ধ্বংস থেকে রক্ষা করে।

## স্লাইড ৩ — তথ্য নিরাপত্তার গুরুত্ব

### 🎯 কেন তথ্য নিরাপত্তা গুরুত্বপূর্ণ?

```
        💰 আর্থিক ক্ষতি রোধ
              ↑
🔒 ব্যবসায়িক ← ICT → 👨‍💼 গ্রাহকের আস্থা
   ধারাবাহিকতা  SEC     রক্ষা
              ↓
        ⚖️ আইনি সম্মতি
```

### 🏢 বাস্তব উদাহরণ — Target Breach (২০১৩)

> **ঘটনা:** যুক্তরাষ্ট্রের বিশাল খুচরা কোম্পানি Target-এ ৪০ মিলিয়ন ক্রেডিট কার্ড তথ্য চুরি হয়।
>
> **কারণ:** একটি HVAC ভেন্ডরের দুর্বল অ্যাক্সেস নিয়ন্ত্রণ।
>
> **ক্ষতি:** $১৬২ মিলিয়ন আর্থিক ক্ষতি + ব্র্যান্ড ক্ষতি।
>
> **শিক্ষা:** থার্ড পার্টি ভেন্ডর নিরাপত্তা নিশ্চিত করতে হবে।

> **📢 স্পিকার নোট:** এই গল্পটি বলুন এবং জিজ্ঞেস করুন: "এই ঘটনায় কী কী ভুল হয়েছিল? কীভাবে এটি এড়ানো যেত?"

## স্লাইড ৪ — CIA ট্রায়াড

### 🔺 তথ্য নিরাপত্তার তিনটি স্তম্ভ

```
                    🔒 CONFIDENTIALITY
                    (গোপনীয়তা)
                         /\
                        /  \
                       /    \
                      /      \
                     /  CIA   \
                    /  TRIAD   \
                   /____________\
    🛡️ INTEGRITY              ✅ AVAILABILITY
    (অখণ্ডতা)                  (প্রাপ্যতা)
```

### 📋 বিস্তারিত ব্যাখ্যা

| স্তম্ভ | অর্থ | উদাহরণ | হুমকি |
|---|---|---|---|
| **Confidentiality** (গোপনীয়তা) | শুধু অনুমোদিত ব্যক্তি তথ্য দেখতে পারবে | পাসওয়ার্ড এনক্রিপশন | ডেটা লিক,盗聽 |
| **Integrity** (অখণ্ডতা) | তথ্য সঠিক ও পরিবর্তনহীন থাকবে | ডিজিটাল স্বাক্ষর | ডেটা টেম্পারিং |
| **Availability** (প্রাপ্যতা) | প্রয়োজনে তথ্য সবসময় পাওয়া যাবে | ৯৯.৯% আপটাইম | DDoS আক্রমণ |

> **🔑 মনে রাখুন:** CIA ট্রায়াড হলো সকল নিরাপত্তা নীতির ভিত্তি।

> **📢 স্পিকার নোট:** প্রতিটি স্তম্ভ ব্যাখ্যা করুন। উদাহরণ: "আপনার ব্যাংক অ্যাকাউন্ট তথ্য — গোপনীয়, সঠিক এবং সবসময় অ্যাক্সেসযোগ্য থাকা দরকার।"

## স্লাইড ৫ — সাধারণ সাইবার হুমকি

### ⚠️ প্রধান সাইবার হুমকিসমূহ

```
🦠 MALWARE          📧 PHISHING         🔐 RANSOMWARE
(ক্ষতিকর সফটওয়্যার)  (প্রতারণামূলক ইমেইল)  (মুক্তিপণ ভাইরাস)

🌊 DDoS Attack      👨‍💻 Insider Threat   🕵️ Social Engineering
(সার্ভার অকার্যকর)   (অভ্যন্তরীণ হুমকি)    (মানবিক প্রতারণা)

🔓 SQL Injection    🌐 Man-in-the-Middle  💻 Zero-Day Exploit
(ডেটাবেস আক্রমণ)    (মাঝপথে আক্রমণ)      (অজানা দুর্বলতা)
```

### 📊 হুমকির প্রকারভেদ ও প্রভাব

| হুমকি | সংজ্ঞা | প্রভাব |
|---|---|---|
| **Malware** | ক্ষতিকর সফটওয়্যার যা সিস্টেমে ক্ষতি করে | ডেটা চুরি, সিস্টেম ক্র্যাশ |
| **Phishing** | নকল ইমেইল/ওয়েবসাইট দিয়ে তথ্য চুরি | পাসওয়ার্ড ও অর্থ চুরি |
| **Ransomware** | ডেটা এনক্রিপ্ট করে মুক্তিপণ দাবি | ব্যবসা বন্ধ, আর্থিক ক্ষতি |
| **DDoS** | সার্ভারে অতিরিক্ত ট্র্যাফিক পাঠানো | ওয়েবসাইট/সার্ভিস অকার্যকর |
| **Insider Threat** | কর্মীদের দ্বারা তথ্য ফাঁস | গোপন ডেটা প্রকাশ |

> **📢 স্পিকার নোট:** "এই হুমকিগুলো কোনো দূরের বিষয় নয়। বাংলাদেশেও প্রতিদিন এই ধরনের আক্রমণ হচ্ছে।"

## স্লাইড ৭ — নিরাপত্তা নীতির সংজ্ঞা ও উদ্দেশ্য

### 📜 নিরাপত্তা নীতি কী?

> **সংজ্ঞা:** একটি নিরাপত্তা নীতি (Security Policy) হলো একটি আনুষ্ঠানিক দলিল যা সংগঠনের তথ্য সম্পদ সুরক্ষার জন্য নিয়মাবলী, দায়িত্ব এবং গ্রহণযোগ্য আচরণ নির্ধারণ করে।

### 🎯 নীতির উদ্দেশ্য

| উদ্দেশ্য | ব্যাখ্যা |
|---|---|
| **সুরক্ষা নিশ্চিত করা** | তথ্য সম্পদকে ঝুঁকি থেকে রক্ষা |
| **আইনি সম্মতি** | আইন ও বিধিমালা মেনে চলা |
| **দায়িত্ব নির্ধারণ** | কে কী করবে তা স্পষ্ট করা |
| **সচেতনতা বৃদ্ধি** | কর্মীদের নিরাপদ আচরণ শেখানো |
| **ঘটনা প্রতিরোধ** | সাইবার ঘটনা হ্রাস করা |

### ⚡ মনে রাখুন: একটি ভালো নীতি হলো...

```
✅ স্পষ্ট ও বোধগম্য     ✅ পরিমাপযোগ্য
✅ বাস্তবায়নযোগ্য       ✅ নিয়মিত আপডেটযোগ্য
✅ ঊর্ধ্বতনের অনুমোদিত   ✅ সবার কাছে প্রচারিত
```

```
│              (RAP)              │      Policy            │

```

### 1️⃣ গ্রহণযোগ্য ব্যবহার নীতি (Acceptable Use Policy — AUP)

> **সংজ্ঞা:** প্রতিষ্ঠানের IT রিসোর্স (কম্পিউটার, ইন্টারনেট, ইমেইল) কীভাবে ব্যবহার করা যাবে এবং কী নিষিদ্ধ তা নির্ধারণ করে।

**AUP তে যা থাকে:**
- অনুমোদিত ও অননুমোদিত ব্যবহার
- ইন্টারনেট ব্রাউজিং সীমাবদ্ধতা
- সামাজিক মিডিয়া নীতি
- সফটওয়্যার ইনস্টলেশন নিয়ম
- লঙ্ঘনের পরিণতি

**বাস্তব উদাহরণ:**
```
❌ কর্মস্থলে ব্যক্তিগত বিনোদনের জন্য প্রতিষ্ঠানের ইন্টারনেট ব্যবহার
❌ লাইসেন্সবিহীন সফটওয়্যার ইনস্টল করা
✅ কাজের প্রয়োজনে ইন্টারনেট ব্যবহার
✅ অনুমোদিত সফটওয়্যার ব্যবহার
```

# অধ্যায় ৪: দুর্যোগ পুনরুদ্ধার (Disaster Recovery)

## স্লাইড ১৯ — দুর্যোগ পুনরুদ্ধার কী?

### 🆘 সংজ্ঞা

> **Disaster Recovery (DR)** হলো এমন একটি পরিকল্পিত প্রক্রিয়া যা কোনো বিপর্যয়ের পর IT সিস্টেম, তথ্য ও অবকাঠামো পুনরুদ্ধার করার পদ্ধতি নির্ধারণ করে।

### 🔍 Backup vs Business Continuity vs Disaster Recovery

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  💾 BACKUP                                          │
│  → ডেটার কপি রাখা (সংকীর্ণ ধারণা)                  │
│                                                     │
│  🔄 DISASTER RECOVERY (DR)                          │
│  → IT সিস্টেম ও অবকাঠামো পুনরুদ্ধার               │
│                                                     │
│  🏢 BUSINESS CONTINUITY (BC)                        │
│  → পুরো ব্যবসায়িক কার্যক্রম চালু রাখা              │
│  (DR + মানব সম্পদ + অফিস + প্রক্রিয়া সব মিলিয়ে)   │
│                                                     │
│  BC ⊃ DR ⊃ Backup                                   │
└─────────────────────────────────────────────────────┘
```

> **📢 স্পিকার নোট:** "শুধু ব্যাকআপ নেওয়া মানেই DR নয়। ব্যাকআপ থেকে সিস্টেম পুনরুদ্ধার করতে কত সময় লাগবে সেটাই DR এর মূল বিষয়।"

## স্লাইড ২০ — দুর্যোগের প্রকারভেদ

### ⚠️ বিভিন্ন ধরনের দুর্যোগ

```
┌──────────────────┬────────────────────────────────┐
│  🌪️ প্রাকৃতিক   │ বন্যা, ভূমিকম্প, ঝড়, আগুন     │
├──────────────────┼────────────────────────────────┤
│  👨‍💻 সাইবার      │ র‍্যানসমওয়্যার, DDoS, ডেটা চুরি │
├──────────────────┼────────────────────────────────┤
│  🤷 মানবিক ভুল  │ ভুল কনফিগারেশন, ডেটা মুছে ফেলা │
├──────────────────┼────────────────────────────────┤
│  🔌 অবকাঠামোগত  │ পাওয়ার ফেইলিউর, নেটওয়ার্ক ডাউন │
├──────────────────┼────────────────────────────────┤
│  💻 হার্ডওয়্যার  │ সার্ভার ক্র্যাশ, ডিস্ক ফেইলিউর  │
└──────────────────┴────────────────────────────────┘
```

### 🔴 বাস্তব ঘটনা: Colonial Pipeline (২০২১)

> **ঘটনা:** র‍্যানসমওয়্যার আক্রমণে যুক্তরাষ্ট্রের বৃহত্তম জ্বালানি পাইপলাইন বন্ধ হয়ে যায়।
>
> **প্রভাব:** পূর্ব উপকূলে জ্বালানি সংকট, $৪.৪ মিলিয়ন মুক্তিপণ প্রদান।
>
> **কারণ:** পুরনো পাসওয়ার্ড, MFA এর অভাব, দুর্বল IR পরিকল্পনা।
>
> **শিক্ষা:** CR পরিকল্পনা, নিয়মিত ব্যাকআপ এবং সেগমেন্টেশন অপরিহার্য।

## স্লাইড ২১ — RTO এবং RPO

### ⏱️ দুটি গুরুত্বপূর্ণ পরিমাপ

```
        বিপর্যয়                    পুনরুদ্ধার
           │                           │
    ───────┼───────────────────────────┼────────
           │                           │
           │◄──────── RTO ────────────►│
           │    (Recovery Time         │
           │     Objective)            │
           │                           │
    ◄──────┤                           │
    │  RPO │                           │
    │(Recovery Point Objective)        │
    │                                  │
  শেষ ব্যাকআপ                     সিস্টেম চালু
```

| পরিমাপ | সংজ্ঞা | উদাহরণ |
|---|---|---|
| **RTO** (Recovery Time Objective) | সিস্টেম পুনরুদ্ধারে সর্বোচ্চ সময় | ৪ ঘণ্টা |
| **RPO** (Recovery Point Objective) | সর্বোচ্চ কত পুরনো ডেটা গ্রহণযোগ্য | ২ ঘণ্টা |

> **💡 মনে রাখুন:** RTO ও RPO যত কম, DR সিস্টেম তত ব্যয়বহুল। ব্যবসার চাহিদা বিশ্লেষণ করে সঠিক লক্ষ্য নির্ধারণ করুন।

## স্লাইড ২৩ — DR সাইটের ধরন

### 🏢 Hot Site vs Warm Site vs Cold Site


| পুনরুদ্ধার সময় | মিনিট | ঘণ্টা | দিন/সপ্তাহ |
| বার্ষিক খরচ | সর্বোচ্চ | মাঝারি | সর্বনিম্ন |


### 🌡️ কুলিং সিস্টেম

- **CRAC (Computer Room Air Conditioning):** নির্ভরযোগ্য শীতাতপ নিয়ন্ত্রণ

- **Liquid Cooling:** উচ্চ ঘনত্বের সার্ভারের জন্য

### 🔄 রিডান্ড্যান্সি মডেল

```
N     → একক সিস্টেম (কোনো ব্যাকআপ নেই)
N+1   → এক অতিরিক্ত উপাদান
2N    → সম্পূর্ণ দ্বিগুণ সিস্টেম
2N+1  → দ্বিগুণ + এক অতিরিক্ত
```

## পরিভাষা অভিধান

| ইংরেজি পরিভাষা | বাংলা অর্থ | সংক্ষিপ্ত ব্যাখ্যা |
|---|---|---|
| CIA Triad | CIA ত্রিভুজ | গোপনীয়তা, অখণ্ডতা, প্রাপ্যতা |
| Malware | ক্ষতিকর সফটওয়্যার | সিস্টেমে ক্ষতির উদ্দেশ্যে তৈরি প্রোগ্রাম |
| Phishing | ফিশিং আক্রমণ | প্রতারণামূলক ইমেইল/ওয়েবসাইট |
| Ransomware | মুক্তিপণ ভাইরাস | ডেটা এনক্রিপ্ট করে মুক্তিপণ দাবি |
| DDoS | বিতরণ সেবা অস্বীকার | একযোগে অতিরিক্ত ট্র্যাফিক আক্রমণ |
| Firewall | ফায়ারওয়াল | নেটওয়ার্ক ট্র্যাফিক ফিল্টার |
| VPN | ভার্চুয়াল প্রাইভেট নেটওয়ার্ক | এনক্রিপ্টেড সুরক্ষিত সংযোগ |
| MFA | বহু-স্তর যাচাইকরণ | একাধিক পদ্ধতিতে পরিচয় যাচাই |
| SIEM | নিরাপত্তা তথ্য ও ঘটনা ব্যবস্থাপনা | কেন্দ্রীয় লগ বিশ্লেষণ সিস্টেম |
| RTO | পুনরুদ্ধার সময় লক্ষ্যমাত্রা | সিস্টেম পুনরুদ্ধারের সর্বোচ্চ সময় |
| RPO | পুনরুদ্ধার বিন্দু লক্ষ্যমাত্রা | গ্রহণযোগ্য সর্বোচ্চ ডেটা ক্ষতি |
| Patch | প্যাচ | নিরাপত্তা সংশোধনী আপডেট |
| Zero-Day | শূন্য-দিনের দুর্বলতা | আবিষ্কৃত কিন্তু অপ্যাচকৃত দুর্বলতা |
| Encryption | এনক্রিপশন | তথ্য কোডে রূপান্তর |
| Biometric | জৈব পরিচিতি | আঙুলের ছাপ/চোখের মনি যাচাই |
| Penetration Testing | অনুপ্রবেশ পরীক্ষা | অনুমোদিত হ্যাকিং পরীক্ষা |
| Social Engineering | সামাজিক প্রকৌশল | মানুষকে প্রতারণা করে তথ্য সংগ্রহ |
| Redundancy | রিডান্ড্যান্সি | অতিরিক্ত ব্যাকআপ ব্যবস্থা |
| Hot Site | হট সাইট | সম্পূর্ণ প্রস্তুত DR সাইট |
| Cold Site | কোল্ড সাইট | শুধু স্থান আছে এমন DR সাইট |
